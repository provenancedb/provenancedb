# Contributing to ProvenanceDB

Thank you for your interest in contributing.

## Ways to Contribute

- Architecture discussions
- Specifications
- Schema design
- Query language design
- Storage engine development
- Documentation
- Testing

## Guiding Expectations

Contributions should align with the project's principles:

- Provenance First
- Trust as Data
- Explainability by Default

## Pull Requests

Please:

1. Open an issue or discussion when proposing significant changes.
2. Keep pull requests focused.
3. Include documentation updates where applicable.
4. Maintain backward compatibility whenever possible.

## Design Philosophy

The goal is not to build another general-purpose database.

The goal is to build a provenance-first knowledge database.
